# VK Multi Audio Upload

Расширение для Chromium-браузеров, расширяющее работу с музыкой на vk.com: массовая загрузка MP3, авто-плейлист, скачивание треков и плейлистов, поиск дубликатов, проверка наличия файлов на странице по их именам, кастомизация раскладки и др.

## Возможности

### Загрузка

Перетаскивание группы MP3 в нативный диалог «Добавить аудиозапись»: файлы выстраиваются в очередь и загружаются по одному. Состояние каждого трека отображается отдельно, упавшие можно перезапустить или скопировать их имена в буфер. Очередь можно ставить на паузу, отдельные файлы — отменять.

ID3v2-теги читаются в браузере; при необходимости отсутствующие исполнитель и название извлекаются из имени файла по шаблону `Исполнитель - Название.mp3` и записываются обратно в файл перед загрузкой.

### Авто-плейлист

По завершении очереди расширение может само открыть диалог создания плейлиста, заполнить название и описание из тегов альбома и установить обложку. Источник обложки — файл из настроек или встроенное изображение APIC из первого трека. На обложку накладывается полупрозрачный водяной знак «перезалито» (готовится через canvas).

### Режим проверки

Альтернатива загрузке: пользователь кидает MP3 в ту же область, расширение читает имена/ID3 и проверяет, какие из этих треков уже есть на странице, а каких нет.

- Имена сверяются с треками страницы через нормализацию: нижний регистр, единые тире, удаление `[bracket]` и `(paren)`-меток, отбрасывание хвостов `feat. X / ft. X / при участии X`.
- Сразу после дропа диалог загрузки закрывается (находит нативную кнопку «Закрыть»).
- Страница автоматически сканируется: по умолчанию первые 100 треков (мгновенно, без прокрутки), опционально — всю страницу до конца с подгрузкой виртуализированных строк.
- Результат рисуется в плавающей панели справа: миникарта-столбец со всеми треками страницы (зелёные маркеры = найденные совпадения), три секции списка — «Отсутствуют», «Уже есть» с позицией `#N`, «Все треки страницы».
- Клик по любому маркеру миникарты, треку из «Уже есть» или строки в «Все треки» прокручивает страницу к нужной аудиозаписи и подсвечивает её.
- У каждой секции — своя иконка копирования. Отсутствующие, имеющиеся, все треки в формате `Artist — Title`, а также отдельная кнопка для копирования уникальных имён артистов.

### Поиск дубликатов в плейлисте

Кнопка в попапе плейлиста разворачивает список целиком (включая виртуализированные строки) и помечает дубликаты по паре «исполнитель + название», а также треки, недоступные по данным VK.

Алгоритм:

1. Раскрытие попапа (клик «Показать все») + skroll внутреннего scroll-контейнера попапа, если он есть. Главную страницу не дёргает.
2. Если попап не отдал треки целиком — fallback на VK API (`al_audio.php?act=load_section` с пагинацией по 50).
3. В последнюю очередь — то, что уже отрисовано в DOM.

Справа от попапа появляется панель с миникартой позиций треков и списком недоступных/дубликатов; клик по маркеру/строке прокручивает попап к нужной аудиозаписи. Доступна кнопка копирования.

### Скачивание

- **Отдельный трек** — иконка скачивания в строке трека (появляется при наведении на строку). Получает прямую ссылку через `reload_audio`, скачивает MP3 напрямую, либо тянет HLS-поток посегментно с расшифровкой AES-128 и извлечением аудио из контейнера MPEG-TS.
- **Отмена скачивания** — повторный клик по той же кнопке во время загрузки прерывает скачивание (иконка превращается в красный «стоп»). HLS-цикл проверяет cancel-токен между сегментами, прямой fetch использует AbortController.
- **Весь плейлист** — две кнопки под нативными кнопками в попапе плейлиста: «по одному файлу» через `chrome.downloads` или одним ZIP-архивом (store-метод, без перекомпрессии). Прогресс отображается полоской с фазой и счётчиком.

### Кастомизация раскладки

- **Закрепить сайдбар** — левая колонка VK становится `position: sticky` и следует за прокруткой страницы. Прилипает не к самому верху viewport, а под верхней панелью (высота top bar измеряется динамически — устойчиво к компактным/полным режимам VK). Подвал сайдбара (Блог / Авторам / Информация о контенте) принудительно сбрасывается в обычный поток — без этого VK применяет к нему скрытую sticky-логику и подвал прыгает под навигацию.
- **Смещение контента** — слайдер от −400 до +400px сдвигает основную колонку, сайдбар и контент верхней панели (логотип/поиск/плеер/аватар) синхронно по горизонтали. Сдвиг применяется через `position: relative + left` к общему flex-родителю и к `TopNavigationWrapper__outer`. Не использует `transform` (тот создаёт новый containing block для `position: fixed` модалок, из-за чего они исчезают). Кнопка «↺» — сброс в 0.

## Установка

1. Скачать или склонировать репозиторий.
2. Открыть `chrome://extensions/` и включить режим разработчика.
3. «Загрузить распакованное расширение» — указать папку проекта.
4. Обновить открытые вкладки vk.com.

## Настройки

Открываются по иконке шестерёнки в шапке диалога загрузки.

| Опция | По умолчанию | Назначение |
|---|---|---|
| Режим работы | Загрузка | Переключатель «Загрузка / Проверка» |
| Сканировать всю страницу | выкл. | В режиме «Проверка» сканировать всю страницу вместо первых 100 |
| Авто-плейлист | выкл. | Создавать плейлист после завершения очереди загрузки |
| Обложка | — | Картинка 1000×1000, используется как обложка плейлиста |
| Обложка из ID3 | выкл. | Использовать APIC из первого трека, если своя не задана |
| Авто-метаданные | выкл. | Дополнять отсутствующие теги из имени файла |
| Закрепить сайдбар | выкл. | Левая колонка VK следует за прокруткой страницы |
| Смещение контента | 0 | Горизонтальный сдвиг макета (px) |

Настройки хранятся в `localStorage` под ключом `vmu_settings_v2`.

## Структура

```
manifest.json        MV3-манифест
background.js        service worker, chrome.downloads
injector_early.js    перехват drop на document_start
injected.js          page-context: VK API, React fiber, HLS, ZIP, ID3, load_section
content.js           isolated world: UI, очередь, настройки, режим проверки, layout
style.css            стили
```

`content.js` и `injected.js` обмениваются сообщениями через `window.postMessage`; всё, что требует доступа к `window.vk`, React fiber и VK CSRF-хешам, выполняется в `injected.js`. `data-vmu-track` стампы на аудиостроках — мост между мирами: injected.js пишет, content.js читает (изолированный мир не видит React-фибров, но видит data-attributes).

## Совместимость

Chromium-браузеры (Chrome, Edge и совместимые), десктопная версия vk.com. Мобильная версия `m.vk.com` не поддерживается. Manifest V3, единственный host-permission — `https://vk.com/*`, внешние зависимости отсутствуют.
